Statistical analysis using Python and R¶

  • Python: Statsmodels, scikit-learn
    • NumPy, SciPy: overlapping functionality
  • R: statistician-designed and focused
  • advanced analysis: R has more to offer, more libraries available
  • many comparable libraries between the two
    • basic stats: either useful
    • basic & advanced: R gets edge
    • machine learning: either useful (Python more likely)

(Generalized) Linear Mixed Models¶

  • general case
  • can be implemented, but can be difficult in the real world
  • at times poorly understood/under-specified
  • if you can implement in a defensible manner, great

(Generalized) Linear Mixed Models¶

  • rule of thumb for fixed and random effects
    • fixed: affects mean
    • random: affects variance
    • real world: predictor dies both
  • balanced vs. unbalanced (missing observations vs. missing participation)
  • Subject/Participant is a random effect at the very least
  • discuss other experimental paradigms as they come up

ANOVA¶

  • special case of the special case
  • experimental data often do not meet assumptions
  • not really suggested if you can do better

Classical tests¶

  • what are the questions being asked
  • have you examined the data at all, i.e., distributions, summaries, etc.
  • use appropriate test for the question/data
  • most tests readily available

Regression¶

  • predictors/factors: continuous (or discrete), categorical or both
  • contrasts, significance, output
  • interpretation
  • plotting: overall, component + residual plots; plotting residuals
  • diagnostics: checking significance of predictors, residuals (see also additional via car package)

  • specifying terms

Formula notation¶

  • form: response ~ predictor1 + predctor2 + ....
  • symbols:
    • +: include in model
    • -: delete from model
    • *: include predictors and calculate interactions
    • :: interaction
    • ^: crossing to nth degree
    • |: conditional on (goes in parentheses in R usually)

For a clear explanation, see this PDF

R¶

  • stats-oriented (see video about development) and this post
  • lots of functionality (see CRAN task views)
  • easy to plot/inspect regression outcome/diagnostics
  • 'bleeding edge' of statistical modeling

scikit-learn¶

  • machine-learning oriented
  • still good for 'simple' analyses
  • useful for prediction/classification or more complex problems
  • caret package in R for machine learning

NumPy, SciPy, scikit-learn, pandas vs. R¶

  • speed
  • memory handling
  • basic/universal vs. domain-specific
  • see R vs. Python infographic: same functionality, depends on comfort and what you need done, likely use BOTH in parallel
  • practical: data munging (general) in Python; statistics in R (additional munging as needed); machine learning in either
